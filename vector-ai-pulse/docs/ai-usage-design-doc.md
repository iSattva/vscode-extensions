# AI Usage — VS Code Extension
## Technical Design & Implementation Specification

**Version:** 1.0 (reverse-engineered from v1.0.0 codebase)
**Audience:** Engineer implementing/replicating the extension from scratch
**Status:** Ready for implementation

---

## 1. Overview

** AI Usage** is a self-contained VS Code extension that tracks AI token consumption, costs, and usage trends directly inside VS Code. It is designed for an enterprise environment with a strict privacy posture:

- **All data stays local.** No server, no telemetry. Data is persisted to a single JSON file on the user's machine.
- **Zero external dependencies at runtime.** Only the `vscode` API and Node.js built-ins (`fs`, `path`, `os`) are used.
- **No build toolchain required.** The extension is plain CommonJS JavaScript, packaged into a `.vsix` by a small Python script (no `vsce`, no npm install needed).

### 1.1 Core capabilities

| Feature | Description |
|---|---|
| Automatic Claude Code tracking | Watches `~/.claude/projects/` session JSONL files in real time and ingests token usage per assistant message |
| Historical import | On first activation, scans all existing Claude Code session files and imports past usage |
| Manual logging | Command-driven entry for ChatGPT, Gemini, Copilot Chat, or any other tool (tool name, input/output tokens, cost) |
| Status bar | Live "today" token count and cost, e.g. `$(graph) 4.2K tok $0.07` |
| Dashboard | Webview with 30-day trend chart, cost by tool, cost by model, cost by workspace, recent sessions, pricing table |
| Sidebar panel | Compact activity-bar webview with summary stats |
| CSV export | Full record export for reporting/analysis |
| Copilot detection | Detects whether GitHub Copilot is installed/active (flat-rate seat license — no per-token cost tracked) |

### 1.2 Supported tools matrix

| Tool | Tracking method | Tokens | Cost |
|---|---|---|---|
| Claude Code | Auto (file watcher on `~/.claude/projects/`) | Exact (from JSONL usage blocks) | Calculated from pricing table |
| ChatGPT / Gemini / other | Manual entry command | User-entered | User-entered |
| GitHub Copilot | Detection only (`installed?` / `active?`) | N/A (flat-rate) | N/A |

## 3. Data model & persistence (`src/store.js`)

### 3.1 Store file

- **Path:** `STORE_PATH = path.join(os.homedir(), '.vscode', 'ai-usage.json')`
- Single JSON document, pretty-printed (`JSON.stringify(data, null, 2)`), UTF-8.
- Read/written synchronously (`fs.readFileSync` / `fs.writeFileSync`) — acceptable because writes are small and infrequent.

### 3.2 Schema

```js
const DEFAULT = {
  version: 1,
  records: [],          // usage records (see 3.3)
  seenMessageIds: [],   // dedup of Claude Code API message IDs
  fileOffsets: {}       // { filePath: byteOffset } for incremental JSONL reads
};
```

### 3.3 Usage record shape

Every ingested or manually logged event appends one record:

```js
{
  timestamp: 1752591000000,   // ms epoch
  tool: 'claude-code',        // 'claude-code' | 'chatgpt' | 'gemini' | 'copilot-chat' | free text
  model: 'claude-sonnet-4-6', // model id string ('manual' or 'unknown' when N/A)
  workspace: 'my-repo',       // workspace/project name (derived from session file path or active workspace)
  sessionId: 'abc-123',       // Claude Code session id (or undefined for manual entries)
  inputTokens: 1234,
  outputTokens: 567,
  cacheReadTokens: 8900,      // optional
  cacheCreateTokens: 0,       // optional
  costUsd: 0.0123             // computed via pricing.calculateCost (or user-entered)
}
```

All aggregation code defensively treats token/cost fields as optional (`r.inputTokens || 0`).

### 3.4 Module-level cache & API

```js
let _cache = null;

function load() {
  if (_cache) return _cache;
  try {
    _cache = JSON.parse(fs.readFileSync(STORE_PATH, 'utf8'));
    // Migration: ensure all keys exist (handle older versions)
    _cache.seenMessageIds = _cache.seenMessageIds || [];
    _cache.fileOffsets    = _cache.fileOffsets    || {};
  } catch {
    _cache = { ...DEFAULT, seenMessageIds: [], fileOffsets: {}, records: [] };
  }
  return _cache;
}

function save() {
  const data = load();
  // Cap seenMessageIds to last 50k to prevent unbounded growth
  if (data.seenMessageIds.length > 50_000) {
    data.seenMessageIds = data.seenMessageIds.slice(-50_000);
  }
  fs.writeFileSync(STORE_PATH, JSON.stringify(data, null, 2), 'utf8');
}

function addRecord(record) { const d = load(); d.records.push(record); save(); }

function hasSeenMessage(msgId) { return load().seenMessageIds.includes(msgId); }
function markSeen(msgId)       { load().seenMessageIds.push(msgId); }

function getFileOffset(filePath)         { return load().fileOffsets[filePath] || 0; }
function setFileOffset(filePath, offset) { load().fileOffsets[filePath] = offset; }

function getRecords() { return load().records; }

function isFirstRun() {
  try { fs.accessSync(STORE_PATH); return false; } catch { return true; }
}

function reset() {
  _cache = { ...DEFAULT, seenMessageIds: [], fileOffsets: {}, records: [] };
  save();
}
```

**Implementation notes**

- The fresh-object spread in `catch`/`reset` (rather than `DEFAULT` directly) prevents shared mutable references to the default arrays/objects.
- `markSeen`/`setFileOffset` mutate the cache but do **not** save; callers batch a `save()` at the end of a processing pass (see watcher design) to avoid excessive disk writes.
- `hasSeenMessage` is O(n) over an array capped at 50k — acceptable at this scale. (Optimization option for the implementer: keep a parallel `Set` in memory.)

### 3.5 Query helpers

```js
function startOf(period) {
  const now = new Date();
  if (period === 'day')
    return new Date(now.getFullYear(), now.getMonth(), now.getDate()).getTime();
  if (period === 'week') {          // start of current week (Sunday), local midnight
    const d = new Date(now);
    d.setDate(d.getDate() - d.getDay());
    d.setHours(0, 0, 0, 0);
    return d.getTime();
  }
  if (period === 'month')
    return new Date(now.getFullYear(), now.getMonth(), 1).getTime();
  if (period === 'last30')
    return Date.now() - 30 * 86_400_000;
  return 0;                          // 'all' — epoch
}

function summarise(period) {
  const since   = startOf(period);
  const records = getRecords().filter(r => r.timestamp >= since);
  return {
    inputTokens:  records.reduce((s, r) => s + (r.inputTokens  || 0), 0),
    outputTokens: records.reduce((s, r) => s + (r.outputTokens || 0), 0),
    totalTokens:  records.reduce((s, r) => s + (r.inputTokens || 0) + (r.outputTokens || 0), 0),
    costUsd:      records.reduce((s, r) => s + (r.costUsd || 0), 0),
    sessionCount: new Set(records.map(r => r.sessionId).filter(Boolean)).size,
    byTool:       groupBy(records, 'tool'),
    byModel:      groupBy(records, 'model'),
    byWorkspace:  groupBy(records, 'workspace'),
    count:        records.length
  };
}

function groupBy(records, key) {
  const result = {};
  for (const r of records) {
    const k = r[key] || 'unknown';
    if (!result[k]) result[k] = { inputTokens: 0, outputTokens: 0, totalTokens: 0, costUsd: 0, count: 0 };
    result[k].inputTokens  += r.inputTokens  || 0;
    result[k].outputTokens += r.outputTokens || 0;
    result[k].totalTokens  += (r.inputTokens || 0) + (r.outputTokens || 0);
    result[k].costUsd      += r.costUsd || 0;
    result[k].count        += 1;
  }
  return result;
}

/** Earliest record timestamp, or null if no records. */
function oldestTimestamp() {
  const records = getRecords();
  if (!records.length) return null;
  return Math.min(...records.map(r => r.timestamp));
}

/** Returns daily totals for the last N days, newest last. */
function dailyTrend(days = 30) {
  const result = [];
  const now = new Date();
  for (let i = days - 1; i >= 0; i--) {
    const d     = new Date(now.getFullYear(), now.getMonth(), now.getDate() - i);
    const start = d.getTime();
    const end   = start + 86_400_000;
    const recs  = getRecords().filter(r => r.timestamp >= start && r.timestamp < end);
    result.push({
      date:    d.toISOString().slice(0, 10),
      label:   `${d.getMonth() + 1}/${d.getDate()}`,
      tokens:  recs.reduce((s, r) => s + (r.inputTokens || 0) + (r.outputTokens || 0), 0),
      costUsd: recs.reduce((s, r) => s + (r.costUsd || 0), 0),
      count:   recs.length
    });
  }
  return result;
}

/** Last N records, newest first. */
function recentRecords(n = 50) {
  return [...getRecords()].sort((a, b) => b.timestamp - a.timestamp).slice(0, n);
}

module.exports = {
  load, save, addRecord,
  hasSeenMessage, markSeen,
  getFileOffset, setFileOffset,
  getRecords, isFirstRun, reset,
  summarise, dailyTrend, recentRecords, oldestTimestamp
};
```

## 4. Pricing engine (`src/pricing.js`)

### 4.1 Configuration

Pricing lives in VS Code user settings under key `humanaAiUsage.pricing`, a map of model-id → per-million-token rates (USD):

```jsonc
"humanaAiUsage.pricing": {
  "claude-sonnet-4-6":          { "input": 3.00,  "output": 15.00, "cacheRead": 0.30 },
  "claude-opus-4-6":            { "input": 15.00, "output": 75.00, "cacheRead": 1.50 },
  "claude-haiku-4-5-20251001":  { "input": 0.80,  "output": 4.00,  "cacheRead": 0.08 }
}
```

Defaults ship in `package.json` `contributes.configuration` set to Anthropic's published rates.

### 4.2 Implementation

```js
'use strict';
const vscode = require('vscode');

const FALLBACK = { input: 3.00, output: 15.00, cacheRead: 0.30 };

function getPricing() {
  return vscode.workspace.getConfiguration('humanaAiUsage').get('pricing') || {};
}

function getModelPricing(model) {
  const table = getPricing();
  // Exact match first, then prefix match (e.g. "claude-sonnet-4-6" matches "claude-sonnet-4-6-...")
  if (table[model]) return table[model];
  const key = Object.keys(table).find(k => model.startsWith(k));
  return key ? table[key] : FALLBACK;
}

/**
 * Calculate cost in USD.
 * @param {string} model
 * @param {number} inputTokens        - billed at full input rate
 * @param {number} outputTokens       - billed at output rate
 * @param {number} cacheReadTokens    - billed at ~10% of input (cacheRead rate)
 * @param {number} cacheCreateTokens  - billed at ~125% of input (treated as input here)
 */
function calculateCost(model, inputTokens = 0, outputTokens = 0, cacheReadTokens = 0, cacheCreateTokens = 0) {
  const p = getModelPricing(model);
  const inputRate      = (p.input     || FALLBACK.input)  / 1_000_000;
  const outputRate     = (p.output    || FALLBACK.output) / 1_000_000;
  const cacheReadRate  = (p.cacheRead || p.input * 0.1 || FALLBACK.cacheRead) / 1_000_000;
  const cacheWriteRate = (p.input     || FALLBACK.input) * 1.25 / 1_000_000;

  return (
    inputTokens       * inputRate +
    outputTokens      * outputRate +
    cacheReadTokens   * cacheReadRate +
    cacheCreateTokens * cacheWriteRate
  );
}

module.exports = { calculateCost, getModelPricing, getPricing };
```

**Design notes**

- Prefix matching lets dated model snapshots (e.g. `claude-sonnet-4-6-20260115`) resolve to the base entry.
- Cache-read tokens default to 10% of input rate when `cacheRead` is not configured; cache-creation tokens are billed at 1.25× input, matching Anthropic's 5-minute-cache write multiplier.
- Everything degrades to `FALLBACK` (Sonnet-class rates) so cost is never `NaN`.

## 5. Claude Code ingestion (`src/claude-watcher.js`)

This is the heart of automatic tracking. Claude Code writes one **JSONL file per session** under `~/.claude/projects/<encoded-project-path>/<session-id>.jsonl`. Each line is a JSON object; assistant turns include `message.usage` with exact token counts.

### 5.1 Constants

```js
const CLAUDE_DIR = path.join(os.homedir(), '.claude', 'projects');
```

### 5.2 `processFile(filePath, isHistorical = false) -> number`

Parses a session JSONL file and appends new usage records. Returns the count of records found. Required behavior:

1. **Incremental read.** Read from `store.getFileOffset(filePath)` to EOF (open with `fs.openSync`/`readSync` or slice the buffer). After processing, call `store.setFileOffset(filePath, newByteOffset)`. This makes live updates O(delta), not O(file).
2. **Line-by-line JSON parse.** Skip malformed/partial trailing lines (do not advance the offset past an incomplete final line — only count fully terminated lines).
3. **Filter to assistant messages with usage.** For each parsed object `obj` where `obj.type === 'assistant'` and `obj.message?.usage` exists:
   - **Dedup:** skip if `store.hasSeenMessage(obj.message.id)`; otherwise `store.markSeen(obj.message.id)`. (Claude Code may rewrite/stream-append the same message; the API message ID is the stable dedup key.)
   - Extract:
     - `inputTokens  = usage.input_tokens || 0`
     - `outputTokens = usage.output_tokens || 0`
     - `cacheReadTokens   = usage.cache_read_input_tokens || 0`
     - `cacheCreateTokens = usage.cache_creation_input_tokens || 0`
     - `model = obj.message.model || 'unknown'`
     - `sessionId = obj.sessionId`
     - `timestamp = Date.parse(obj.timestamp) || Date.now()`
     - `workspace` = decoded from the parent directory name of the JSONL file (Claude Code encodes the project path as the folder name with `/` → `-`; take the last path segment for display).
   - Compute `costUsd = pricing.calculateCost(model, inputTokens, outputTokens, cacheReadTokens, cacheCreateTokens)`.
   - `store.addRecordNoSave(...)` — see note below — with `tool: 'claude-code'`.
4. **Do not save per record.** The caller batches `store.save()`. (In the reference code, `processFile` pushes records into the loaded cache; `scanHistorical` and the watcher handler call `store.save()` once per pass.)

### 5.3 `scanHistorical(onProgress) -> Promise<number>`

First-run import of all existing sessions:

```js
async function scanHistorical(onProgress) {
  // Recursively enumerate CLAUDE_DIR for **/*.jsonl into `files`
  let total = 0;
  for (let i = 0; i < files.length; i++) {
    onProgress && onProgress(i + 1, files.length);
    total += processFile(files[i], true);
  }
  store.save();
  return total;
}
```

- Wrap in `vscode.window.withProgress({ location: Notification, title: 'Importing Claude Code history…' })` from `extension.js`, forwarding `onProgress(done, total)` to the progress reporter.
- Must be resilient to `CLAUDE_DIR` not existing (return 0).

### 5.4 `startWatcher(onChange) -> Disposable`

Live tracking (verbatim reference logic):

```js
/**
 * Start watching ~/.claude/projects for live changes.
 * Returns a disposable.
 */
function startWatcher(onChange) {
  if (!fs.existsSync(CLAUDE_DIR)) return { dispose: () => {} };

  // VS Code file watcher — covers new files and changes recursively
  const pattern = new vscode.RelativePattern(
    vscode.Uri.file(CLAUDE_DIR),
    '**/*.jsonl'
  );
  const watcher = vscode.workspace.createFileSystemWatcher(pattern, false, false, true);
  //                                       ignoreCreate=false, ignoreChange=false, ignoreDelete=true

  const handle = (uri) => {
    const found = processFile(uri.fsPath);
    if (found > 0) {
      store.save();
      onChange && onChange(found);
    }
  };

  watcher.onDidCreate(handle);
  watcher.onDidChange(handle);

  return watcher;
}

module.exports = { scanHistorical, startWatcher, processFile };
```

`onChange(foundCount)` is used by `extension.js` to refresh the status bar and any open dashboard/sidebar webviews.

## 8. Extension entry point (`extension.js`)

Not fully captured in screenshots; specified here to match the README's observable behavior.

### 8.1 Activation flow

```js
function activate(context) {
  // 1. Status bar item (right side), command = open dashboard
  statusItem = vscode.window.createStatusBarItem(vscode.StatusBarAlignment.Right, 100);
  statusItem.command = 'humanaAiUsage.openDashboard';
  refreshStatusBar();          // "$(graph) 4.2K tok $0.07" from summarise('day')
  statusItem.show();

  // 2. First run -> historical import with progress notification
  if (store.isFirstRun()) {
    vscode.window.withProgress(
      { location: vscode.ProgressLocation.Notification, title: 'Humana AI Usage: importing Claude Code history' },
      (progress) => scanHistorical((done, total) =>
        progress.report({ message: `${done}/${total} sessions` }))
    ).then(n => vscode.window.showInformationMessage(`Imported ${n} usage records.`));
  }

  // 3. Live watcher
  context.subscriptions.push(startWatcher(() => { refreshStatusBar(); refreshWebviews(); }));

  // 4. Commands + webview view provider (sidebar)
  //    - humanaAiUsage.openDashboard
  //    - humanaAiUsage.logManualEntry
  //    - humanaAiUsage.exportCsv
  //    - (optional) humanaAiUsage.reset
}
```

### 8.2 Commands (all reachable via `Ctrl+Shift+P`)

**`Humana AI Usage: Open Dashboard`** — creates (or reveals) a `vscode.window.createWebviewPanel('humanaAiUsage', 'Humana AI Usage', ViewColumn.One)`, sets `panel.webview.html = buildDashboard(data)`. Re-render on watcher `onChange` while the panel is open.

**`Humana AI Usage: Log Manual Entry`** — sequential `showQuickPick`/`showInputBox` prompts: tool (chatgpt / gemini / copilot-chat / other), input tokens, output tokens, cost (optional; if blank and tool has pricing, compute; else 0). Appends record with `model: 'manual'`, `workspace` = active workspace folder name, `timestamp: Date.now()`, then `store.save()` + refresh.

**`Humana AI Usage: Export to CSV`** — `vscode.window.showSaveDialog({ filters: { CSV: ['csv'] } })`, then write header `timestamp,date,tool,model,workspace,sessionId,inputTokens,outputTokens,cacheReadTokens,cacheCreateTokens,costUsd` plus one row per record (escape commas/quotes).

### 8.3 Status bar format

`$(graph) {fmtTok(today.totalTokens)} tok {fmt$(today.costUsd)}` with tooltip showing today/week/month cost breakdown. Refresh on: activation, watcher events, manual entry, config change (`onDidChangeConfiguration` for `humanaAiUsage.pricing`).

### 9.2 `build-vsix.py`

A `.vsix` is just a ZIP with a specific layout: `[Content_Types].xml` and `extension.vsixmanifest` at the root, and all extension files under `extension/`.

```python
"""
build-vsix.py -- Packages ai-usage into a .vsix without vsce.
Run: python build-vsix.py
"""
import json, zipfile, textwrap, pathlib

ROOT = pathlib.Path(__file__).parent
with open(ROOT / "package.json") as f:
    pkg = json.load(f)

NAME      = pkg["name"];        VERSION = pkg["version"]
PUBLISHER = pkg["publisher"];   DISPLAY = pkg["displayName"]
DESC      = pkg["description"]; ENGINE  = pkg["engines"]["vscode"]
OUT       = ROOT / f"{NAME}-{VERSION}.vsix"

INCLUDE = [
    "package.json", "extension.js", "README.md",
    "images/icon.png", "images/activity-icon.svg",
    "src/store.js", "src/claude-watcher.js", "src/copilot-tracker.js",
    "src/pricing.js", "src/dashboard.js",
]

CONTENT_TYPES = textwrap.dedent("""\
    <?xml version="1.0" encoding="utf-8"?>
    <Types xmlns="http://schemas.openxmlformats.org/package/2006/content-types">
      <Default Extension="vsixmanifest" ContentType="text/xml" />
      <Default Extension="json" ContentType="application/json" />
      <Default Extension="js"   ContentType="application/javascript" />
      <Default Extension="md"   ContentType="text/markdown" />
      <Default Extension="png"  ContentType="image/png" />
      <Default Extension="svg"  ContentType="image/svg+xml" />
    </Types>
    """)

MANIFEST = textwrap.dedent(f"""\
    <?xml version="1.0" encoding="utf-8"?>
    <PackageManifest Version="2.0.0"
        xmlns="http://schemas.microsoft.com/developer/vsx-schema/2011"
        xmlns:d="http://schemas.microsoft.com/developer/vsx-schema-design/2011">
      <Metadata>
        <Identity Language="en-US" Id="{NAME}" Version="{VERSION}" Publisher="{PUBLISHER}" />
        <DisplayName>{DISPLAY}</DisplayName>
        <Description xml:space="preserve">{DESC}</Description>
        <Tags>{",".join(pkg.get("keywords", []))}</Tags>
        <Categories>{",".join(pkg.get("categories", ["Other"]))}</Categories>
        <GalleryFlags>Public</GalleryFlags>
        <Properties>
          <Property Id="Microsoft.VisualStudio.Code.Engine" Value="{ENGINE}" />
        </Properties>
      </Metadata>
      <Installation>
        <InstallationTarget Id="Microsoft.VisualStudio.Code" Version="[1.73.0, )" />
      </Installation>
      <Dependencies />
      <Assets>
        <Asset Type="Microsoft.VisualStudio.Code.Manifest"
            Path="extension/package.json" Addressable="true" />
        <Asset Type="Microsoft.VisualStudio.Services.Content.Details"
            Path="extension/README.md" Addressable="true" />
        <Asset Type="Microsoft.VisualStudio.Services.Icons.Default"
            Path="extension/images/icon.png" Addressable="true" />
        <Asset Type="Microsoft.VisualStudio.Services.Icons.Small"
            Path="extension/images/icon.png" Addressable="true" />
      </Assets>
    </PackageManifest>
    """)

with zipfile.ZipFile(OUT, "w", compression=zipfile.ZIP_DEFLATED) as zf:
    zf.writestr("[Content_Types].xml", CONTENT_TYPES)
    zf.writestr("extension.vsixmanifest", MANIFEST)
    for rel in INCLUDE:
        src = ROOT / rel
        if not src.exists():
            raise FileNotFoundError(f"Missing: {rel}")
        zf.write(src, f"extension/{rel}")

size_kb = OUT.stat().st_size / 1024
print(f"Built:    {OUT.name} ({size_kb:.1f} KB)")
print(f"Contents: {', '.join(INCLUDE)}")
print(f"Install:  Extensions panel > ... > Install from VSIX")
```

## 10. Key design decisions & rationale

1. **Local-only JSON store** — satisfies enterprise data-governance constraints (no PHI/PII leaves the machine; nothing to security-review beyond the file watcher).
2. **Byte-offset incremental reads + message-ID dedup** — the two mechanisms are complementary: offsets make live tailing cheap; `seenMessageIds` guarantees idempotency across historical re-scans, file rewrites, streaming partial appends, and `reset()`+rescan scenarios.
3. **Deferred `save()`** — mutating helpers don't persist; each processing pass persists once. Keeps disk churn low during rapid session streaming.
4. **Prefix-matched pricing with fallback** — resilient to new dated model snapshots without config changes.
5. **No bundler/vsce** — plain CommonJS + Python zip packaging means the whole thing builds on a locked-down corporate laptop with only Python and VS Code installed.
6. **SVG-in-HTML charts** — avoids shipping a JS chart library into the webview (smaller attack surface, no CSP exceptions beyond inline styles).

## 11. Edge cases the implementation must handle

- `~/.claude/projects` absent (Claude Code not installed) → watcher no-ops, historical scan returns 0.
- Corrupt/older store file → `load()` catch path resets to defaults without crashing.
- Partial trailing JSONL line during streaming → skip; don't advance offset past it.
- File truncated/rotated (offset > file size) → reset that file's offset to 0 and reprocess (dedup prevents double counting).
- Records missing token/cost fields → all reducers use `|| 0`.
- `seenMessageIds` growth → capped at 50,000 on every `save()`.
- Webview HTML injection → every user-derived string goes through `escHtml`.

## 12. Acceptance criteria / test plan

1. **First run:** install VSIX with existing Claude Code history → progress notification, records imported once; second reload imports nothing new.
2. **Live tracking:** run a Claude Code session → status bar updates within seconds of each assistant turn; dashboard trend reflects today's usage.
3. **Dedup:** delete `fileOffsets` from the store and reload → re-scan produces zero duplicate records.
4. **Pricing:** change `humanaAiUsage.pricing` for a model → new records use new rates; dashboard pricing table reflects settings.
5. **Manual entry:** log a ChatGPT entry → appears in byTool breakdown and recent table.
6. **CSV export:** exported row count equals `store.getRecords().length`; opens cleanly in Excel.
7. **Copilot chip:** with Copilot installed, dashboard shows installed/active status.
8. **Privacy:** with a network monitor attached, extension makes zero outbound requests.
